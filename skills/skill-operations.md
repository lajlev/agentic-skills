# Skill Operations

Use this workflow to keep skills aligned with user goals.

## During Onboarding
1. Extract repeated tasks from user answers.
2. Match each task to an existing installed skill.
3. For uncovered tasks, choose one path:
   - install curated skill
   - install skill from GitHub repo
   - create a local custom skill
4. Validate each new skill on one real task.
5. Record all changes in `skills/skills-registry.md`.

## During Normal Usage
1. Detect repeated requests that are slow or manual.
2. Propose a skill improvement when coverage is weak.
3. After approval, install or create the skill.
4. Update `skills/skills-registry.md` and `AGENTS.md` if behavior changes.

## Quality Bar For Skills
- Clear trigger conditions.
- Clear workflow steps.
- Reusable scripts/references when needed.
- Demonstrated benefit on at least one real task.
